# Grazioso Salvare Animal Rescue Dashboard
**Computer Science Portfolio Reflection**

**Writing Maintainable, Readable, and Adaptable Programs**
Writing maintainable and adaptable software requires strict adherence to modular design principles, specifically the Model-View-Controller (MVC) architecture. In this project, I achieved this by completely separating the database operations from the front-end dashboard logic. The `AnimalShelter` Python module I engineered serves as the data controller, encapsulating all PyMongo CRUD operations (Create, Read, Update, Delete) behind clean, predictable methods. By keeping the database connection strings, error handling, and query execution inside this standalone module, the dashboard code (`ProjectTwoDashboard.ipynb`) remains remarkably clean. The primary advantage of this architecture is reusability; the UI layer does not need to know how MongoDB works, it only needs to pass query dictionaries to the `read()` method. Because this CRUD module is completely decoupled from the Plotly Dash front-end, I can easily drop it into future Python applications, RESTful APIs, or automated backend scripts that require secure, authenticated access to the MongoDB dataset without rewriting any database logic.

**Approaching Problems as a Computer Scientist**
My approach to solving technical problems focuses on dissecting the client’s operational needs and mapping them directly to the appropriate technological stack. For Grazioso Salvare, the requirement was to instantly identify dogs matching highly specific Search and Rescue (SAR) profiles from an unstructured dataset of 10,000 animal outcomes. Unlike previous academic assignments that focused on isolated algorithms, this project required full-stack systems integration. I evaluated the data and selected MongoDB to handle the unstructured, schema-less records natively, and Plotly Dash to build a reactive, Python-native interface. A critical part of my strategy was anticipating and neutralizing data pipeline issues—such as intercepting and stripping out MongoDB's default `ObjectID` fields to prevent downstream JSON serialization crashes in the Dash DataTable. Moving forward, I will continue to leverage this strategy: deeply analyzing the shape of the client's data first, selecting databases that natively support that shape, and building robust middleware to handle the data transformations safely.

**The Role and Impact of Computer Scientists**
Computer scientists bridge the gap between raw data and actionable intelligence. We design the systems that allow organizations to cut through massive amounts of information to execute their missions effectively. For a company like Grazioso Salvare, manually sorting through 10,000 shelter records across five distinct locations to find a 26-week-old intact male German Shepherd for Mountain Rescue is a logistical bottleneck. By developing this interactive dashboard, I transformed a static database into a dynamic targeting tool. Staff can now execute complex, multi-parameter queries with a single click, instantly visualizing the geographical locations and breed distributions of viable SAR candidates. This directly accelerates their operational capabilities, saving the company time and resources, and ultimately allowing them to train more rescue animals to save human lives.
